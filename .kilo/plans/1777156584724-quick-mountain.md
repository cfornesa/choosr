# Plan: Fix "New Artwork" Button Behavior

## Problem Statement
The "New Artwork" button on studio.php is not functioning correctly. When clicked, it should act like a page refresh, clearing the current artwork state and preparing the canvas for a completely new artwork. Instead, clicking this button multiple times and then saving results in updating an already-loaded artwork rather than creating a new one.

## Root Cause Analysis
1. The `_newArtworkBtn` DOM element is referenced in `app.js` (line 976) but **no event listener is ever attached** to it
2. Without a click handler, the button does nothing when clicked
3. The `_currentArtworkId` variable is never cleared, so subsequent saves continue to PATCH the existing artwork instead of POSTing a new one
4. The metadata panel retains the previous artwork's data (title, description, etc.)
5. The canvas is not cleared/reset

## Assumption Surfaced (Rule 1)
**Assumption:** Clicking "New Artwork" should reset the application to a "fresh canvas" state where:
- The current artwork ID is cleared (so next save creates a new artwork, not updates existing)
- Metadata fields (title, description, tags) are cleared
- The canvas is cleared/reset with default visual dimensions
- The mode remains the same (Manual or Data-driven)
- Dataset selection remains (for convenience)
- Style selection remains (for convenience)

**Question to confirm:** Should the "New Artwork" button also clear the dataset selection and reset style to default, or should it preserve those for convenience while only clearing the artwork identity?

## Options (Rule 2)

### Option A: Full Reset Handler (Recommended)
Create a dedicated `_onNewArtworkClick` handler that:
1. Clears `_currentArtworkId` (so save creates new artwork)
2. Calls `_clearArtworkMetadata()` to clear title/description/tags
3. Calls `Controls.reset()` to clear canvas and reset to defaults
4. Shows status message "New artwork started"

**Pros:** Clean separation of concerns, consistent behavior
**Cons:** None significant

### Option B: Page Reload Simulation
Simulate a page refresh by reloading studio.php:
```javascript
window.location.reload();
```

**Pros:** Absolutely guaranteed fresh state
**Cons:** Loses unsaved work without warning, slower UX, unnecessary server request
**Reframe:** This violates the principle of least surprise - users expect in-app buttons to work in-app, not trigger page reloads

### Option C: Minimal Fix (Unexpected Alternative)
Only clear the `_currentArtworkId` and metadata, leave everything else as-is:
```javascript
function _onNewArtworkClick() {
  _currentArtworkId = null;
  _clearArtworkMetadata();
  _showStatus('Ready to create new artwork');
}
```

**Pros:** Minimal code change, preserves user's current creative settings
**Cons:** Canvas retains previous visual, which may confuse users about whether "new" actually happened
**User signal alignment:** Based on user's description of "act like a page refresh", they likely expect visual reset too

## Recommended Solution: Option A (Full Reset Handler)

This best matches the user's expectation of "act like a page refresh" while:
- Preserving the in-app experience (no actual page reload)
- Clearing all artwork identity (ID, metadata)
- Resetting visual state to defaults
- Providing clear user feedback via status message

## Implementation Details

### Changes to `src/app.js`:

1. Add event listener in `init()` function (after other button listeners):
```javascript
if (_newArtworkBtn) {
  _newArtworkBtn.addEventListener('click', _onNewArtworkClick);
}
```

2. Create new handler function `_onNewArtworkClick`:
```javascript
function _onNewArtworkClick() {
  log('Starting new artwork');
  
  // Clear artwork ID so save creates new, not updates
  _currentArtworkId = null;
  if (_currentArtworkIdInput) {
    _currentArtworkIdInput.value = '';
  }
  
  // Clear metadata fields
  _clearArtworkMetadata();
  
  // Reset controls (clears canvas, resets to defaults)
  if (window.DataToArt && window.DataToArt.Controls) {
    window.DataToArt.Controls.reset();
  }
  
  // Update delete button visibility (hidden when no artwork loaded)
  _updateDeleteButtonVisibility();
  
  // Show status to confirm action
  _showStatus('New artwork started - canvas reset');
}
```

3. Note: The `_clearArtworkMetadata()` function already exists (lines 568-580) and clears all metadata fields plus sets `_currentArtworkId = null`. We may need to adjust it slightly to not call `Controls.reset()` if that's not desired, or keep the call if full reset is appropriate.
