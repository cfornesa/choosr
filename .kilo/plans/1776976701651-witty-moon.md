## Schema Overview

The database supports:
1. **User authentication and galleries** (Phase 2)
2. **Dataset storage** from uploads, preloaded sources, and API feeds
3. **Column-level metadata** for the visual mapping UI
4. **Artwork state persistence** including full creative configuration
5. **API response caching** with TTL
6. **Extensible art style enumeration**

---

## Table Specifications

### 1. `users`
**Purpose:** User accounts for authentication and gallery ownership.

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | Surrogate key |
| username | VARCHAR(64) | UNIQUE, NOT NULL | Display name |
| email | VARCHAR(255) | UNIQUE, NOT NULL | Login credential |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt hash |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Account creation |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | Last modified |
| is_active | TINYINT(1) | DEFAULT 1 | Soft delete flag |

**Indexes:**
- PRIMARY KEY (id)
- UNIQUE KEY (username)
- UNIQUE KEY (email)
- KEY (created_at) вҖ” for admin queries

---

### 2. `datasets`
**Purpose:** Stores normalized data from uploads, preloaded sources, or API feeds. Includes sanitization status per C-04.

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | Surrogate key |
| user_id | INT | FOREIGN KEY вҶ’ users(id), ON DELETE CASCADE | Owner (NULL for preloaded) |
| source_type | ENUM('upload', 'preloaded', 'api_feed') | NOT NULL | Data origin |
| source_name | VARCHAR(255) | NOT NULL | Filename or dataset label |
| original_filename | VARCHAR(255) | NULL | Original upload name (if applicable) |
| file_size_bytes | INT UNSIGNED | NULL | File size for uploads |
| mime_type | VARCHAR(100) | NULL | MIME type from upload |
| storage_path | VARCHAR(500) | NULL | Server path to normalized data |
| row_count | INT UNSIGNED | DEFAULT 0 | Number of data rows |
| is_sanitized | TINYINT(1) | DEFAULT 0 | **C-04: Sanitization status** |
| sanitized_at | DATETIME | NULL | When sanitization completed |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Record creation |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | Last modified |

**Indexes:**
- PRIMARY KEY (id)
- KEY (user_id) вҖ” for user's dataset list
- KEY (source_type) вҖ” for filtering by source
- KEY (created_at) вҖ” for chronological queries
- KEY (is_sanitized) вҖ” C-04: queries pending sanitization

---

### 3. `dataset_columns`
**Purpose:** Column-level metadata for each dataset, enabling the column mapper UI to display data types and display names.

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | Surrogate key |
| dataset_id | INT | FOREIGN KEY вҶ’ datasets(id), ON DELETE CASCADE | Parent dataset |
| column_name | VARCHAR(128) | NOT NULL | Original column header |
| display_name | VARCHAR(128) | NOT NULL | User-friendly label for UI |
| data_type | ENUM('string', 'number', 'date', 'boolean', 'unknown') | DEFAULT 'unknown' | Detected type |
| sample_values | JSON | NULL | Array of sample values for preview |
| column_order | TINYINT UNSIGNED | NOT NULL | Display order (0-indexed) |
| is_mappable | TINYINT(1) | DEFAULT 1 | Can this column map to visual dims? |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Record creation |

**Indexes:**
- PRIMARY KEY (id)
- KEY (dataset_id) вҖ” for loading columns with dataset
- KEY (column_order) вҖ” for display ordering

---

### 4. `artworks`
**Purpose:** Persisted artwork state including column mappings, style choices, palettes, and rendering configuration.

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | Surrogate key |
| user_id | INT | FOREIGN KEY вҶ’ users(id), ON DELETE CASCADE | Owner |
| dataset_id | INT | FOREIGN KEY вҶ’ datasets(id), ON DELETE SET NULL | Source dataset (nullable if deleted) |
| title | VARCHAR(255) | NOT NULL | Artwork name |
| description | TEXT | NULL | Optional notes |
| art_style_id | INT | FOREIGN KEY вҶ’ art_styles(id), ON DELETE RESTRICT | Chosen style (prevent delete if in use) |
| column_mapping | JSON | NOT NULL | Mapping: { visual_dim: column_id } |
| palette_config | JSON | NOT NULL | Selected/custom palette definition |
| rendering_config | JSON | NULL | Additional renderer parameters |
| is_public | TINYINT(1) | DEFAULT 0 | Gallery visibility |
| thumbnail_path | VARCHAR(500) | NULL | Cached preview image path |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Creation time |
| updated_at | DATETIME | ON UPDATE CURRENT_TIMESTAMP | Last modified |

**Indexes:**
- PRIMARY KEY (id)
- KEY (user_id) вҖ” for user's gallery
- KEY (dataset_id) вҖ” for finding artworks by data source
- KEY (art_style_id) вҖ” for filtering by style
- KEY (is_public, created_at) вҖ” for public gallery feed
- KEY (created_at) вҖ” for chronological queries

---

### 5. `api_cache`
**Purpose:** Cached API feed responses with TTL timestamp to avoid repeated external requests.

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | Surrogate key |
| source_url | VARCHAR(1000) | NOT NULL | Full API endpoint URL |
| source_name | VARCHAR(255) | NOT NULL | Human-readable label |
| response_data | JSON | NOT NULL | Cached JSON response |
| cached_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | When cached |
| expires_at | DATETIME | NOT NULL | TTL expiration timestamp |
| etag | VARCHAR(255) | NULL | HTTP ETag for conditional refresh |
| last_modified | VARCHAR(255) | NULL | HTTP Last-Modified header |
| http_status | SMALLINT | DEFAULT 200 | Response status code |
| access_count | INT UNSIGNED | DEFAULT 0 | Usage statistics |
| last_accessed_at | DATETIME | NULL | Last read time |

**Indexes:**
- PRIMARY KEY (id)
- UNIQUE KEY (source_url) вҖ” prevent duplicate caches
- KEY (expires_at) вҖ” for cache invalidation queries
- KEY (cached_at) вҖ” for oldest-first cleanup
- KEY (source_name) вҖ” for admin/library UI

---

### 6. `art_styles`
**Purpose:** Enumerated rendering modes. Extensible table for new styles.

| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | Surrogate key |
| style_key | VARCHAR(64) | UNIQUE, NOT NULL | Machine identifier (e.g., 'particle_field') |
| display_name | VARCHAR(128) | NOT NULL | Human-readable name |
| description | TEXT | NULL | Style explanation |
| default_config | JSON | NULL | Default rendering parameters |
| preview_svg | TEXT | NULL | Icon/preview markup |
| is_active | TINYINT(1) | DEFAULT 1 | Show in selector? |
| sort_order | TINYINT UNSIGNED | DEFAULT 0 | Display position |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Record creation |

**Indexes:**
- PRIMARY KEY (id)
- UNIQUE KEY (style_key) вҖ” prevent duplicates
- KEY (is_active, sort_order) вҖ” for style selector query

**Seed Data (to be included as INSERT statements):**
1. `particle_field` вҖ” "Particle Field" вҖ” Floating particles with properties mapped to data
2. `geometric_grid` вҖ” "Geometric Grid" вҖ” Structured grid patterns with data-driven geometry
3. `flowing_curves` вҖ” "Flowing Curves" вҖ” Organic curve flows with data influence

## Technical Requirements

| Requirement | Implementation |
|-------------|----------------|
| Engine | InnoDB for all tables |
| Charset | utf8mb4 with utf8mb4_unicode_ci collation |
| Primary Keys | Auto-increment INT on all tables |
| Foreign Keys | Explicit ON DELETE CASCADE or SET NULL |
| FK Indexes | All foreign key columns indexed |
| Query Indexes | Columns in WHERE clauses indexed (user_id, dataset_id, source_type, created_at, etc.) |
| Idempotency | CREATE TABLE IF NOT EXISTS throughout |
| Seeding | art_styles table seeded with 3 initial modes via INSERT |
