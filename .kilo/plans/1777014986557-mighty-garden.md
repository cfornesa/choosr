# Data-to-Art Studio Auth UI + Art Styles Repair Plan

## Problem Analysis

### Issue 1: Auth Panel Regression
When logged in to `/studio.php`, the Account section incorrectly shows login inputs instead of "Logged in as USERNAME" + Logout button.

**Root Cause:** The CSS-based visibility toggle depends on the `.dta-auth-logged-in` class being present on `#dta-auth-section`, but the JavaScript initialization logic has a timing/ordering issue. The `_authState` detection relies on `data-authenticated` attribute from the sidebar, but the `_updateAuthUI()` function applies the class to the auth section. The CSS rules exist but may not be triggered properly due to how the auth state is being read from DOM attributes.

**Current Code Issues:**
- `studio.php` hardcodes `data-authenticated="1"` on sidebar for all authenticated users (correct)
- `app.js` checks `sidebarEl.dataset.authenticated === '1'` but also checks `sidebarEl.dataset.username` which could be empty string (falsy)
- The PHP outputs `data-username` with `$_SESSION['username']` but the login API sets `$_SESSION['user_id']` and `$_SESSION['email']`, not `username`
- Empty string username causes auth detection to fail in some code paths

### Issue 2: Art Style Select Empty
The ART STYLE dropdown shows "— Select art style —" with no options.

**Root Cause:** The `ArtStyles.listStyles()` returns an empty array because the style registry is populated during the artStyles.js IIFE execution, which happens before the individual style modules (particleField.js, etc.) have executed and attached themselves to `window.DataToArt`. The script load order appears correct in HTML, but the registration logic checks for styles at load time, not when they're actually available.

**Current Code Issues:**
- `artStyles.js` IIFE checks for `window.DataToArt.ParticleFieldStyle` etc. immediately on load
- If style modules haven't executed yet, registry stays empty
- `app.js` init() finds `window.DataToArt.ArtStyles` exists but `listStyles()` returns `[]`
- No error is shown, select just stays empty

## Implementation Plan

### 1. Auth State Detection + UI Switching (app.js)

**Change:** Fix the auth detection logic to properly handle the data attributes.

- **Lines 635-645:** Modify the auth state detection to:
  - Check `data-authenticated="1"` explicitly (which studio.php always sets for auth'd users)
  - Fall back to checking if `data-username` is non-empty for index.php compatibility
  - Ensure `_authState.username` gets a default value if empty
  
- **Function `_updateAuthUI()`:** Ensure it properly:
  - Adds/removes `.dta-auth-logged-in` class on `#dta-auth-section`
  - Updates `_authStatus` text element
  - Updates the summary text via `#dta-auth-summary`
  - Login form and logout button visibility handled by existing CSS

### 2. Art Styles Population (app.js + artStyles.js)

**Change A (artStyles.js):** Add a delayed/retried registration mechanism.

- Add a `registerBuiltinStyles()` function that attempts to register styles
- Call it immediately (for already-loaded styles) and also expose it for delayed calls
- Keep backward compatibility with existing auto-registration

**Change B (app.js):** Add defensive delayed initialization for art styles.

- In `init()`, after the initial style population attempt, check if any styles were found
- If not, set a short timeout (100ms) to retry, giving scripts time to finish executing
- Also add a manual call to force re-check the global namespace for style modules

### 3. CSS Visibility Rules (app.css)

**No changes required** - The existing CSS rules are correct:
- `#dta-auth-section.dta-auth-logged-in #dta-auth-forms-container { display: none !important; }`
- `#dta-auth-section:not(.dta-auth-logged-in) #dta-logout-btn { display: none; }`

These rules properly hide the login form when logged in and hide the logout button when logged out.

### 4. Controls Initialization (app.js)

**Verify:** Ensure `Controls.init()` is called with correct parameters and that `#dta-controls` container is properly passed. The current code already does this correctly at lines 652-655:

```javascript
window.DataToArt.Controls.init(_canvasEl, _controlsEl, {
  styleKey: 'particleField',
  renderingConfig: { animate: false }
});
```

The ColumnMapper and PalettePicker are rendered into containers created inside Controls.init(), which are appended to `_controlsEl` (#dta-controls). This should work once the styles are properly populated.
