# Plan: Fix Art Style Centering - Coordinate System Mismatch

## Problem Analysis

The previous fix attempted to add offset to styles, but this created a double-application issue. The root cause is a coordinate system mismatch:

### Renderer Transform (lines 617-631)
```javascript
var translateX = (normX - 0.5) * cssWidth;   // offset from X dimension
var translateY = (normY - 0.5) * cssHeight;  // offset from Y dimension

this.ctx.translate(cssWidth / 2 + translateX, cssHeight / 2 + translateY);
this.ctx.rotate(rotationRad);
this.ctx.scale(scaleFactor, scaleFactor);
```

This moves the coordinate origin to **canvas center plus X/Y offset**. After this transform:
- `(0, 0)` = canvas center (adjusted by X/Y sliders)
- `(-w/2, -h/2)` = top-left of canvas
- `(w/2, h/2)` = bottom-right of canvas

### Current barCode Positioning (WRONG)
```javascript
var startX = (w - totalBarsWidth) / 2;  // Centers on absolute canvas, ignores transform
var startY = (h - barHeight) / 2;
```

When origin is at canvas center, `(w - totalBarsWidth) / 2` places the barcode far right of center because it assumes top-left origin.

### particleField Pattern (CORRECT)
```javascript
// In manual mode, origin is at canvas center - position relative to that
px = (normX - 0.5) * drawW;  // Centers at origin when normX = 0.5
py = (normY - 0.5) * drawH;
```

## Solution

Remove the offset addition from the previous fix and instead position elements relative to the transformed origin (0, 0 = canvas center).

### For barCode
```javascript
// OLD (wrong - assumes top-left origin)
var startX = (w - totalBarsWidth) / 2;
var startY = (h - barHeight) / 2;

// NEW (correct - positions relative to transformed origin at center)
var startX = -totalBarsWidth / 2;
var startY = -barHeight / 2;
```

### For scatterMatrix
```javascript
// OLD (wrong)
var startX = (w - matrixW) / 2;
var startY = (h - matrixH) / 2;

// NEW (correct)
var startX = -matrixW / 2;
var startY = -matrixH / 2;
```

### For voronoiCells
The point generation uses `(p.x - 0.5) * width` which centers points at origin. BUT the sampling loop in `_drawVoronoiFromPoints` goes from `(0, 0)` to `(w, h)`:

```javascript
for (var y = 0; y < h; y += 4) {
  for (var x = 0; x < w; x += 4) {
    // samples and draws
  }
}
```

After the canvas transform, `(0, 0)` is at canvas center, so the loop only covers the bottom-right quadrant! The fix is to sample from `(-w/2, -h/2)` to `(w/2, h/2)`:

```javascript
// OLD - samples from origin (canvas center) to bottom-right
for (var y = 0; y < h; y += 4)
for (var x = 0; x < w; x += 4)

// NEW - samples full canvas centered at origin
for (var y = -h/2; y < h/2; y += 4)
for (var x = -w/2; x < w/2; x += 4)
```
