## Part 2: src/app.js Architecture

### Structure
- Wrapped in IIFE: `(function() { 'use strict'; ... })();`
- Exposes minimal namespace: `window.DataToArt.App`
- DEBUG flag checked from URL: `window.location.search.indexOf('debug=true') !== -1`
- All console.log gated by DEBUG

### Initialization Flow (DOMContentLoaded)

1. **Grab DOM References**
   - canvasEl = document.getElementById('dta-canvas')

   ```javascript
   var styles = window.DataToArt.ArtStyles.listStyles();
   ```

4. **Fetch Initial Dataset List**
   - GET ../api/datasets.php
   - Populate datasetSelect with options: value = dataset_id, text = source_name
   - Handle errors via _showError()

### Event Wiring

#### File Upload
```javascript
uploadInput.addEventListener('change', function(e) {
  var file = e.target.files[0];
  if (!file) return;
  
  var formData = new FormData();
  formData.append('file', file);
  
  fetch('../api/upload.php', {
    method: 'POST',
    body: formData
  })
  .then(handleResponse)
  .then(function(data) {
    // Refresh dataset list
    loadDatasetList();
    // Auto-select newly uploaded dataset
    datasetSelect.value = data.dataset_id;
    // Trigger dataset load
    loadDataset(data.dataset_id);
  })
  .catch(_showError);
});
```

#### Dataset Selection
```javascript
datasetSelect.addEventListener('change', function(e) {
  var datasetId = e.target.value;
  if (!datasetId) return;
  loadDataset(datasetId);
});

function loadDataset(datasetId) {
  // For Phase 1: datasets.php GET returns full list with columns
  // We can reuse the already-fetched list or refetch
  // Implementation option: refetch to ensure fresh data
  fetch('../api/datasets.php')
    .then(handleResponse)
    .then(function(data) {
      var dataset = data.datasets.find(function(d) { return d.id == datasetId; });
      if (dataset) {
        // Transform via DataMapper
        var mapped = window.DataToArt.DataMapper.mapApiResponse({
          dataset_id: dataset.id,
          columns: dataset.columns,
          row_count: dataset.row_count
        });
        // Load into controls
        window.DataToArt.Controls.loadDataset(mapped);
      }
    })
    .catch(_showError);
}
```

#### Style Selection
```javascript
styleSelect.addEventListener('change', function(e) {
  var styleKey = e.target.value;
  if (styleKey) {
    window.DataToArt.Controls.setStyle(styleKey);
  }
});
```

#### Render & Export Buttons
```javascript
renderBtn.addEventListener('click', function() {

exportBtn.addEventListener('click', function() {

```

### Auth Implementation (Minimal Phase 1)

**Register Flow:**
```javascript
// On register form submit
fetch('../api/auth/register.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username, email, password })
})
.then(handleResponse)
.then(function(data) {
  _showStatus('Registration successful. Please log in.');
  // Switch to login form
})
.catch(function(err) {
  _showError(err.message || 'Registration failed');
});
```

**Login Flow:**
```javascript
// On login form submit
fetch('../api/auth/login.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email, password })
})
.then(handleResponse)
.then(function(data) {
  _showStatus('Logged in as ' + data.username);
  _currentUsername = data.username;
  // Refresh dataset list to get user-owned datasets
  loadDatasetList();
})
.catch(function(err) {
  _showError(err.message || 'Login failed');
});
```

### Helper Functions

```javascript
// Standard response handler
function handleResponse(response) {
  if (!response.ok) {
    return response.json().then(function(data) {
      throw new Error(data.error || 'Request failed');
    });
  }
  return response.json();
}

// Error display (inline, minimal)
function _showError(message) {
  if (DEBUG) console.error('[DataToArt.App]', message);
  errorDisplay.textContent = message;
  errorDisplay.style.display = 'block';
  setTimeout(function() {
    errorDisplay.style.display = 'none';
  }, 5000);
}

// Status display (success/info)
function _showStatus(message) {
  if (DEBUG) console.log('[DataToArt.App]', message);
  errorDisplay.textContent = message;
  errorDisplay.style.display = 'block';
  errorDisplay.style.color = '#4a8fa8'; // teal accent for success
  setTimeout(function() {
    errorDisplay.style.display = 'none';
    errorDisplay.style.color = ''; // reset
  }, 3000);
}
```

## Gallery of Options

### Option A: Sidebar Layout (Recommended)
Controls as a fixed-width sidebar on the right. Canvas takes remaining space. Matches DESIGN.md "canvas as hero" principle with controls as "instruments around it."

### Option B: Floating Panel Layout
Controls as floating panels over/around canvas. More space-efficient but less structured. May conflict with "atelier workstation" metaphor.

### Option C: Bottom Drawer Layout
Controls in a collapsible bottom drawer. Better for smaller screens but hides controls by default. Conflicts with "controls as instruments" visibility.

**Recommended:** Option A — sidebar layout honors the atelier metaphor, keeps controls visible and accessible, and follows the established desktop-first constraint.
