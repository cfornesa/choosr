# Plan: Data-to-Art Studio — Account Placement + Logout UX

## Assumption to Surface
**The user is always authenticated when viewing studio.php** (it redirects to `/` otherwise). This means we can simplify the Account section to show logout/status immediately, rather than showing login/register forms that will never be used on this page. The current code treats studio.php as if it might show auth forms to anonymous users, but the PHP redirect prevents this.

---

## 1. Sidebar Layout Strategy

### Current Order (problematic)
1. Upload/Dataset/Style controls
2. Dynamic controls (ColumnMapper/PalettePicker)
3. **Account section (last)** ← too hidden

### Proposed Order
1. **Account section (top)** — shows "Logged in as USERNAME" + Logout
2. Upload/Dataset/Style controls
3. Dynamic controls (ColumnMapper/PalettePicker)

### HTML Changes in studio.php
- Move the entire `<details id="dta-auth-section">` block from its current position (after `#dta-controls`) to immediately after the `<aside id="dta-sidebar">` opening tag.
- No wrapper changes needed — keep the Account section as a distinct block at the top.
- Add an ID to the logged-in status text container for easier JS targeting.

## 2. Logout UX Design

### Placement
- Inside the Account block, below the status text.
- Button: `<button id="dta-logout-btn">Log Out</button>` (already exists, just needs visibility).

### Behavior
- **Fetch**: POST to `api/auth/logout.php` (already implemented in JS).
- **On success**: Clear `_currentUsername`, show status, redirect to `/index.php` after 800ms (consistent with existing pattern).
- **On error**: Display via existing `_showError()` mechanism.

### JS Wiring in src/app.js
- The event listener already exists (line 526-528).
- The `_onLogoutClick` handler already exists (lines 343-378) and handles redirect.
- **Minimal change**: Ensure the logout button is visible by default in studio.php (since users are always authenticated here).

## 4. Auth State Clarity

### Studio.php Behavior (authenticated-only page)
Since studio.php redirects unauthenticated users to `/`:
- **Always show**: "Logged in as USERNAME" status + Logout button.
- **Never show**: Login/register forms (they're irrelevant here).

### Implementation approach:
1. **PHP session check on page load**: Use the existing session to populate a `data-username` attribute or inline text showing the current user.
2. **JS initialization**: On `init()`, read the username and update `_currentUsername`, `_authStatus.textContent`, and `_logoutBtn.style.display = 'block'` immediately.
3. **Keep existing endpoints**: No changes to `api/auth/session.php`, `login.php`, or `logout.php` contracts.

### Auth flow preservation:
- Logout → POST to `api/auth/logout.php` → redirect to `/index.php`.
- Next studio.php visit requires re-authentication (PHP handles redirect).

## Gallery of Options (Rule 2)

### Option A: Minimal — Just move and style (Recommended)
Move Account section to top, add teal border, ensure logout visible. Keeps existing collapsible `<details>` behavior.

### Option B: Reframe — Strip Account to essentials
Since studio.php is auth-only, remove the `<details>` wrapper entirely. Show a compact "Logged in as X | Log Out" header-style bar at the top of the sidebar. This challenges the premise that we need a collapsible Account section on an authenticated-only page.

### Option C: Unexpected — Persistent top bar
Move Account out of sidebar entirely. Create a thin top bar below `#dta-header` that spans full width, showing user status and logout. Makes auth state globally visible, not sidebar-bound.

**Recommendation**: Option A. It respects the existing design language while solving the visibility problem with minimal change.
