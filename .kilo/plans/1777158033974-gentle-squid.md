# Plan: Fix Footer Overlap on portfolio.php

## Problem Summary
The final row of portfolio content renders directly on top of the footer instead of above it, obscuring the footer content.

## Root Cause Analysis

1. **Duplicate CSS selectors**: The `#dta-portfolio-main` selector is defined twice in the CSS:
   - Lines 59-64: Includes `flex: 1 1 auto` (correct)
   - Lines 183-188: Missing `flex: 1 1 auto`, adds `min-height: calc(100vh - 140px)` (incorrect)
   
   The second definition overrides the first, breaking the flex layout.

2. **Body uses flex column layout** (lines 38-42):
   ```css
   body {
     min-height: 100vh;
     display: flex;
     flex-direction: column;
   }
   ```
   For this to work properly, the main content area needs `flex: 1 1 auto` to expand and push the footer down.

3. **Hardcoded min-height calculation** (`calc(100vh - 140px)`) is brittle and doesn't account for dynamic content or exact header/footer dimensions.

## Solution Options

### Option 1: Consolidate CSS and Fix Flex Layout (Recommended)
Merge the two `#dta-portfolio-main` definitions, keeping `flex: 1 1 auto` and removing the problematic `min-height` calculation:

```css
#dta-portfolio-main {
  flex: 1 1 auto;           /* Allow main to grow and push footer down */
  max-width: 1200px;
  margin: 0 auto;
  padding: 32px 24px;
  width: 100%;              /* Ensure it takes full width within max-width constraint */
}
```

**Pros:**
- Uses CSS flexbox properly to push footer to bottom
- Handles dynamic content of any height
- Simpler, more maintainable code
- No magic numbers

**Cons:**
- Removes the explicit minimum height (not actually needed with flex)

### Option 2: Keep Both Properties
Consolidate both definitions but keep `flex: 1 1 auto` and add `min-height`:

```css
#dta-portfolio-main {
  flex: 1 1 auto;
  max-width: 1200px;
  margin: 0 auto;
  padding: 32px 24px;
  min-height: calc(100vh - 200px);  /* Adjusted for header + footer */
}
```

**Pros:**
- Ensures some minimum content height
- Still uses flexbox

**Cons:**
- Magic number calculation still brittle
- May cause scrollbar issues on short pages

### Option 3: Add Footer Padding/Margin (Reframe)
Instead of fixing the main content, add sufficient padding to the footer to ensure visibility:

```css
#dta-portfolio-footer {
  padding: 48px 24px;  /* Increased from 24px */
  margin-top: 48px;
  /* ... */
}
```

**Pros:**
- Minimal changes to existing structure
- Ensures footer content visibility

**Cons:**
- Doesn't address root cause (broken flex layout)
- Content may still visually overlap if not enough space

## Recommendation
**Option 1** is the cleanest solution. The `flex: 1 1 auto` property is designed exactly for this use case - it allows the main content to expand and push the footer to the bottom of the viewport or below the content, whichever is taller. The `min-height` calculation is unnecessary when flexbox is working correctly.
