# Plan: Fix Art Style Rendering Issues in Manual Mode

## Investigation Summary

The user reported four specific rendering issues in studio.php when using Manual Dimensions mode:

1. **Geometric Grid** appears centered around (-0.75, -0.25) instead of canvas center
2. **Voronoi Cells** appears centered around (-0.75, -0.8) instead of canvas center  
3. **Scatter Matrix** appears very small by default
4. **Bar Code** looks like dotted paper with short vertical lines instead of actual bar code

## Root Cause Analysis

### Issue 1 & 2: Centering Problems (Geometric Grid & Voronoi Cells)

**Root Cause:** The `renderUsingExplicitDimensions` method in `renderer.js` applies canvas-level transformations (translate, rotate, scale) to center the artwork, and sets `renderConfig.manualMode = true`. However:

- **GeometricGrid** does NOT check for `manualMode` flag - it uses the raw data point coordinates directly for grid cell calculation without the `(p.x - 0.5) * width` adjustment needed for centered coordinate systems.

- **VoronoiCells** DOES check for `manualMode` and uses `(p.x - 0.5) * width` positioning, but there appears to be a conflict with how the Voronoi cells are being rendered that offsets them incorrectly.

The canvas transform moves the origin to center, but styles need to interpret coordinates relative to that new origin.

### Issue 3: Scatter Matrix Too Small

**Root Cause:** In `scatterMatrix.js` manual mode:
- `cellSize = ((p.size || 0.5) * MAX_SIZE) * 0.08` = 200 * 0.5 * 0.08 = **8px cells** (extremely small)
- `rows = Math.max(2, Math.floor(((p.size || 0.5) * MAX_SIZE) * 0.02))` = max(2, 2) = **2 rows**

The scaling factors (0.08 and 0.02) were designed for iterating all 30 data points, but the result is a tiny 2x2 matrix of 8px cells - visually insignificant.

### Issue 4: Bar Code Looks Like Dotted Paper

**Root Cause:** In `barCode.js` manual mode:
- `barCount = Math.max(3, Math.floor(((p.size || 0.5) * MAX_SIZE) * 0.02))` = ~3 bars per data point
- Bars are drawn at scattered positions based on each data point's (x, y) coordinates
- With 30 data points × 3 bars each = 90 tiny bars scattered randomly across canvas

The result resembles dotted paper with short lines rather than a cohesive bar code pattern.

## Proposed Solutions

### Solution for Issues 1 & 2: Fix Centering in Manual Mode

**Option A (Recommended):** Update GeometricGrid to handle manualMode flag like other styles
- Add manualMode check in geometricGrid.js render function
- When manualMode is true, use `(p.x - 0.5) * width` style positioning instead of grid-based positioning
- This makes GeometricGrid consistent with VoronoiCells, particleField, etc.

**Option B:** Remove canvas-level transforms from renderer and handle positioning in each style
- More work, but eliminates the transform complexity
- Each style fully controls its own coordinate system

**Selected: Option A** - Follows established pattern in other styles, minimal changes

### Solution for Issue 3: Fix Scatter Matrix Size

Increase the scaling factors and improve the grid calculation:
- Change `cellSize` factor from `0.08` to `0.25` (25% of MAX_SIZE instead of 8%)
- Change `rows` calculation to produce a reasonable grid (4-6 rows instead of 2)
- Draw a single cohesive scatter matrix instead of one per data point, OR
- Reduce number of matrices drawn to avoid clutter

**Selected:** Draw a single scatter matrix for all data points (like data-driven mode) but using manual dimension parameters for styling.

### Solution for Issue 4: Fix Bar Code Rendering

**Option A:** Redesign barCode manual mode to draw one cohesive bar code pattern
- Draw bars in a single horizontal line across canvas center
- Use data points to determine bar heights/patterns (not positions)
- Actually looks like a bar code

**Option B:** Keep scattered pattern but increase bar size and reduce count
- Still looks like dotted paper, just bigger dots

**Selected: Option A** - Create proper bar code visualization

## Implementation Plan

### Phase 1: Fix Geometric Grid Centering
**File:** `src/canvas/styles/geometricGrid.js`
- Add manualMode detection from renderingConfig
- When manualMode=true, calculate cell positions using centered coordinates
- Map the 30 data points to grid cells with proper centering

### Phase 2: Fix Voronoi Cells Centering
**File:** `src/canvas/styles/voronoiCells.js`

- Ensure Voronoi cells are centered properly around canvas center

### Phase 3: Fix Scatter Matrix Size
**File:** `src/canvas/styles/scatterMatrix.js`
- Redesign manual mode to draw one cohesive matrix
- Use reasonable cell sizes (20-40px instead of 8px)

### Phase 4: Fix Bar Code Rendering
**File:** `src/canvas/styles/barCode.js`
- Redesign manual mode to draw one cohesive bar code
- Draw horizontal bars across canvas center
- Use data points to vary bar width/height/color, not position
- Create recognizable bar code aesthetic

## Assumptions Surfaced

1. **Manual mode should produce visually similar output to data-driven mode** - Users expect the same art style to look consistent regardless of mode.

2. **Canvas transforms are the correct architecture** - The renderer's approach of using canvas-level transforms is sound; styles just need to handle the centered coordinate system properly.

3. **Bar code should look like a traditional barcode** - Linear pattern of vertical bars, not scattered elements.

4. **Scatter matrix should be a single cohesive visualization** - Not 30 tiny matrices scattered across the canvas.
