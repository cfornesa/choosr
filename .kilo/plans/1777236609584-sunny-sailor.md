# Mobile CSS Fixes Plan

## Issues Identified

### 1. exhibit.php `<main>` shows ~2x screen width on mobile
**Assumption:** The `#dta-exhibit-main` max-width constraint is being bypassed or its parent container isn't properly constraining it on small screens.

**Solution:** Add mobile-specific override in `css/app.css` at the existing `max-width: 768px` media query to ensure `#dta-exhibit-main` uses `max-width: 100%` with proper `padding: 16px` and doesn't overflow.

### 2. exhibit.php `<main>` width should match portfolio.php
**Assumption:** The user wants consistent width behavior between the two pages.

**Solution:** In the 768px media query, set `#dta-exhibit-main` to use `max-width: 100%` similar to how portfolio handles it (portfolio uses `max-width: 1200px` on desktop; on mobile it should be fluid).

### 3. exhibit.php "All Artworks" link should show only arrow on mobile
**Assumption:** The user finds the full text distracting in mobile view and prefers just the arrow.

**Solution:** In `css/app.css` at the 768px media query, add a rule to hide the text content of `#dta-exhibit-header a` and show only `::before` pseudo-element with the arrow character. Alternatively, use `text-indent: -9999px` with a background arrow, or set `font-size: 0` on the text while showing the arrow via pseudo-element.

### 4. Hamburger menu icon centered instead of right-aligned on mobile
**Assumption:** The hamburger is inside `.dta-header-title` which uses `flex: 1 1 auto` on mobile (line 976 in app.css), and the `justify-content: space-between` on `#dta-header` isn't pushing it to the far right because `.dta-header-title` is flex with `gap: 24px`.

**Solution:** In the 768px media query, add `.dta-hamburger { margin-left: auto; }` to push the hamburger to the right edge of the header.

### 5. studio.php configuration panels (sidebar) should be below canvas at 100% width on mobile
**Assumption:** On mobile, the current side-by-side layout of canvas and sidebar needs to stack vertically with the sidebar full-width below the canvas.

**Solution:** In the 768px media query, change `#dta-main` to `flex-direction: column` and make `#dta-sidebar` full-width (`flex: none; width: 100%`).

## Proposed CSS Changes (additions to `css/app.css` within existing `@media (max-width: 768px)` block)

```css
@media (max-width: 768px) {
  /* Existing rules... */

  /* Issue 1 & 2: exhibit.php main width */
  #dta-exhibit-main {
    max-width: 100%;
    padding: 16px;
  }

  /* Issue 3: hide text, show arrow only */
  #dta-exhibit-header a {
    font-size: 0;
    letter-spacing: 0;
  }
  #dta-exhibit-header a::before {
    content: "←";
    font-size: 16px;
    font-weight: 700;
  }

  /* Issue 4: hamburger right-aligned */
  .dta-hamburger {
    margin-left: auto;
  }

  /* Issue 5: studio.php sidebar stacks below canvas */
  #dta-main {
    flex-direction: column;
    height: auto;
    overflow: auto;
  }

  #dta-canvas-region {
    flex: none;
    width: 100%;
    height: 60vh;
    min-height: 300px;
  }

  #dta-sidebar {
    flex: none;
    width: 100%;
    max-width: none;
    min-width: 0;
    border-left: none;
    border-top: 2px solid #c9922a;
  }
}
```

## Feasibility Assessment

All 5 fixes are **pure CSS changes** — no HTML modifications required.

- Issue 1, 2, 3, 4: Simple additions to existing media query
- Issue 5: Minor layout reflow using flex-direction

**Recommendation:** Proceed with implementation. All changes are low-risk, targeted CSS edits.
