## File 1: api/datasets.php

### Supported HTTP Methods
| Method | Operation | Description |
|--------|-----------|-------------|
| GET    | List datasets | Return all datasets available to the current user |
| DELETE | Delete dataset | Remove a user-owned dataset and its physical file |

### Request/Response Formats

**GET Request:**
- No query parameters required
- Headers: Cookie with session (required)

**GET Response (200):**
```json
{
  "success": true,
  "datasets": [
    {
      "id": 1,
      "user_id": 5,
      "source_type": "upload",
      "source_name": "sales_data",
      "original_filename": "sales_q1.csv",
      "file_size_bytes": 15420,
      "mime_type": "text/csv",
      "storage_path": "/uploads/uuid.csv",
      "row_count": 500,
      "is_sanitized": 1,
      "sanitized_at": "2026-04-23 12:00:00",
      "created_at": "2026-04-23 10:00:00",
      "updated_at": "2026-04-23 10:05:00",
      "columns": [
        {
          "id": 1,
          "column_name": "revenue",
          "display_name": "Revenue",
          "data_type": "number",
          "sample_values": ["1000", "2500", "3000"],
          "column_order": 0,
          "is_mappable": 1
        }
      ]
    }
  ]
}
```

**DELETE Request:**
- Query string: `?id={dataset_id}`
- Headers: Cookie with session (required)

**DELETE Response (200):**
```json
{ "success": true }
```

**Error Response (400/401/404/500):**
```json
{ "success": false, "error": "Error message" }
```

### Auth Requirements
- **GET**: Requires session — include `require_once __DIR__ . '/auth/session.php'` at top
- **DELETE**: Requires session — same include

### Database Queries

**GET — Fetch datasets:**
```sql
SELECT d.* 
FROM datasets d 
WHERE d.user_id = ? OR d.source_type = 'preloaded' 
ORDER BY d.created_at DESC
```

**GET — Fetch columns for each dataset:**
```sql
SELECT id, column_name, display_name, data_type, sample_values, column_order, is_mappable
FROM dataset_columns 
WHERE dataset_id = ? 
ORDER BY column_order ASC
```

**DELETE — Verify ownership:**
```sql
SELECT user_id, storage_path 
FROM datasets 
WHERE id = ? AND user_id = ?
```

**DELETE — Delete record:**
```sql
DELETE FROM datasets WHERE id = ? AND user_id = ?
```

### Error Handling Strategy
1. **Method not allowed**: Return 405 with `{"success": false, "error": "Method not allowed"}`
2. **Session failure**: session.php handles 401 and exit — no additional code needed
3. **DELETE missing id**: Return 400 with `{"success": false, "error": "Dataset ID required"}`
4. **DELETE invalid id**: Return 400 with `{"success": false, "error": "Invalid dataset ID"}`
5. **DELETE not found/unauthorized**: Return 404 with `{"success": false, "error": "Dataset not found"}`
6. **File deletion failure**: Log error but still return success (DB is source of truth)
7. **Database errors**: Return 500 with generic message (debug detail if APP_DEBUG)

## File 2: api/apiFeeds.php

### Supported HTTP Methods
| Method | Operation | Description |
|--------|-----------|-------------|
| GET    | Fetch API feed | Retrieve external API data with caching |

### Request/Response Formats

**GET Request:**
- Query string: `?source_url={url}&source_name={name}`
- Both parameters required
- Headers: Cookie with session (required)

**GET Response (200) — Cached:**
```json
{
  "success": true,
  "data": { /* cached response */ },
  "cached": true,
  "cached_at": "2026-04-23 10:00:00"
}
```

**GET Response (200) — Fresh:**
```json
{
  "success": true,
  "data": { /* fetched response */ },
  "cached": false,
  "cached_at": "2026-04-23 12:00:00"
}
```

**Error Response (400/401/404/500):**
```json
{ "success": false, "error": "Error message" }
```

### Auth Requirements
- Requires session — include `require_once __DIR__ . '/auth/session.php'` at top

### Database Queries

**Check cache:**
```sql
SELECT response_data, cached_at, access_count
FROM api_cache
WHERE source_url = ? AND expires_at > NOW()
```

**Update access count (on cache hit):**
```sql
UPDATE api_cache
SET access_count = access_count + 1,
    last_accessed_at = NOW()
WHERE source_url = ?
```

**Insert/update cache (on cache miss):**
```sql
INSERT INTO api_cache 
(source_url, source_name, response_data, cached_at, expires_at, etag, last_modified, http_status)
VALUES (?, ?, ?, NOW(), DATE_ADD(NOW(), INTERVAL ? SECOND), ?, ?, ?)
ON DUPLICATE KEY UPDATE
    source_name = VALUES(source_name),
    response_data = VALUES(response_data),
    cached_at = VALUES(cached_at),
    expires_at = VALUES(expires_at),
    etag = VALUES(etag),
    last_modified = VALUES(last_modified),
    http_status = VALUES(http_status),
    access_count = access_count + 1,
    last_accessed_at = NOW()
```

### External Operations (cURL)
- Timeout: 10 seconds (CURLOPT_TIMEOUT)
- Follow redirects: Yes (CURLOPT_FOLLOWLOCATION)
- Return transfer: Yes (CURLOPT_RETURNTRANSFER)
- Extract headers: Yes (CURLOPT_HEADERFUNCTION or separate header retrieval)
- Extract ETag and Last-Modified headers for storage

### Error Handling Strategy
1. **Missing parameters**: Return 400 with `{"success": false, "error": "source_url and source_name are required"}`
2. **Invalid URL**: Return 400 with `{"success": false, "error": "Invalid source_url"}` (use filter_var FILTER_VALIDATE_URL)
3. **Session failure**: session.php handles 401
4. **cURL failure**: Return 500 with `{"success": false, "error": "Failed to fetch API feed"}` (log actual curl error internally, do not expose in response)
5. **Invalid JSON in response**: Return 500 with `{"success": false, "error": "Invalid JSON response from API"}`
6. **Database errors**: Return 500 with generic message (debug detail if APP_DEBUG)

## File 3: api/artwork.php

### Supported HTTP Methods
| Method | Operation | Description |
|--------|-----------|-------------|
| POST   | Save artwork | Create a new artwork record |
| GET    | Retrieve artwork(s) | Get single artwork or list user's artworks |
| DELETE | Delete artwork | Remove an artwork and its thumbnail |

### Request/Response Formats

**POST Request:**
- Body: JSON
```json
{
  "dataset_id": 1,
  "art_style_id": 2,
  "title": "My Artwork",
  "column_mapping": {"x": "revenue", "y": "profit"},
  "palette_config": {"primary": "#c9922a", "secondary": "#4a8fa8"},
  "rendering_config": {"speed": 0.5, "density": 100},
  "is_public": 0
}
```
- Headers: Cookie with session (required), Content-Type: application/json
- dataset_id is nullable (can be null)
- is_public defaults to 0 if not provided

**POST Response (201):**
```json
{
  "success": true,
  "artwork_id": 42
}
```

**GET Request (single artwork):**
- Query string: `?id={artwork_id}`
- Headers: Cookie with session (optional — public artworks accessible without session)

**GET Request (list all):**
- No query parameters
- Headers: Cookie with session (required)

**GET Response (single artwork, 200):**

```json
{
```

**GET Response (list, 200):**

**DELETE Request:**
- Query string: `?id={artwork_id}`
- Headers: Cookie with session (required)

**DELETE Response (200):**
```json
{ "success": true }
```

**Error Response (400/401/403/404/500):**
```json
{ "success": false, "error": "Error message" }
```

### Auth Requirements
- **POST**: Requires session — include `require_once __DIR__ . '/auth/session.php'` at top
- **GET (with id)**: Does NOT require session for public artworks; requires session only for private artworks owned by the user
- **GET (list all)**: Requires session — include session.php
- **DELETE**: Requires session — include session.php

### Database Queries

**POST — Validate art_style_id exists:**
```sql
SELECT id FROM art_styles WHERE id = ? AND is_active = 1
```

**POST — If dataset_id provided, verify ownership or preloaded:**
```sql
SELECT id, user_id, source_type FROM datasets WHERE id = ?
```

**POST — Insert artwork:**
```sql
INSERT INTO artworks 
(user_id, dataset_id, art_style_id, title, column_mapping, palette_config, rendering_config, is_public)
VALUES (?, ?, ?, ?, ?, ?, ?, ?)
```

**GET — Single artwork (with ownership check):**
```sql
SELECT a.*, s.display_name as art_style_name
FROM artworks a
JOIN art_styles s ON a.art_style_id = s.id
WHERE a.id = ? AND (a.is_public = 1 OR a.user_id = ?)
```

**GET — List user's artworks:**
```sql
SELECT a.*, s.display_name as art_style_name
FROM artworks a
JOIN art_styles s ON a.art_style_id = s.id
WHERE a.user_id = ?
ORDER BY a.created_at DESC
```

**DELETE — Verify ownership:**
```sql
SELECT user_id, thumbnail_path FROM artworks WHERE id = ? AND user_id = ?
```

**DELETE — Delete record:**
```sql
DELETE FROM artworks WHERE id = ? AND user_id = ?
```

### Error Handling Strategy
1. **Method not allowed**: Return 405 with error message
2. **Session failure**: session.php handles 401 (except for public artwork GET)
3. **POST invalid JSON body**: Return 400 with `{"success": false, "error": "Invalid JSON body"}`
4. **POST missing required fields**: Return 400 with specific field names
5. **POST title too long**: Return 400 with `{"success": false, "error": "Title must not exceed 255 characters"}`
6. **POST invalid art_style_id**: Return 400 with `{"success": false, "error": "Invalid art style"}`
7. **POST invalid dataset_id**: Return 400 with `{"success": false, "error": "Dataset not found or access denied"}`
8. **POST invalid column_mapping/palette_config**: Return 400 with `{"success": false, "error": "column_mapping must be a valid JSON object"}`
9. **GET single not found/unauthorized**: Return 404 with `{"success": false, "error": "Artwork not found"}`
10. **DELETE missing id**: Return 400 with error message
11. **DELETE not found/unauthorized**: Return 404 with error message
12. **Thumbnail deletion failure**: Log error but return success (DB is source of truth)
13. **Database errors**: Return 500 with generic message (debug detail if APP_DEBUG)
