## Problem Statement
The artwork titles in exhibit.php display as "1" instead of the actual title. The user reports this line is problematic:
```php
<h1><?php echo htmlspecialchars($artwork['title'] || 'Untitled'); ?></h1>
```

## Root Cause Analysis
In PHP, the `||` operator is a **logical OR** that returns a boolean (`true` or `false`), not the first truthy value like JavaScript.

When PHP evaluates `$artwork['title'] || 'Untitled'`:
1. If `$artwork['title']` exists and is truthy, the expression returns `true`
2. If `$artwork['title']` is falsy/empty, PHP evaluates `'Untitled'` (which is truthy) and returns `true`
3. The result is always boolean `true`, which when echoed displays as "1"

## Solution
Replace the `||` operator with a ternary using `!empty()` check. This handles both `null` and empty string cases.

The `!empty()` check returns:
- `false` if the variable is not set, null, empty string, "0", 0, false, or empty array
- `true` otherwise (variable has a value)

Example fix:
```php
// BEFORE (buggy - outputs "1")
<?php echo htmlspecialchars($artwork['title'] || 'Untitled'); ?>

// AFTER (correct - outputs actual title or "Untitled")
<?php echo htmlspecialchars(!empty($artwork['title']) ? $artwork['title'] : 'Untitled'); ?>
```
