**Date:** 2026-04-23  
**Target:** public/css/app.css (primary), minimal HTML class additions  
**Goal:** Refine CSS to achieve calm gallery first impression with playful lab interactions

## Plan Overview

This plan refines the existing CSS to achieve two emotional states:
1. **On load:** Calm, intentional gallery composition — the user trusts the space
2. **On interaction:** Subtle playful lab qualities emerge through tactile feedback

---

## 1. Global Adjustments

### Intent
Create visual breathing room and establish a calmer, more intentional rhythm throughout the interface.

### Changes

**Typography refinements:**
- Increase base `font-size` from `14px` to `15px` for improved readability
- Adjust `line-height` from `1.5` to `1.6` for better text breathing room
- Add subtle letter-spacing to body for a more refined feel
- Ensure monospace text (data readouts) has adequate contrast and spacing

**Spacing rhythm:**
- Increase `#dta-sidebar` padding from `16px` to `20px` for less cramped feel
- Increase gap between control groups from `16px` to `20px`
- Add consistent vertical rhythm using multiples of 4px

**Selectors to touch:**
```css
html, body { font-size, line-height }
#dta-sidebar { padding, gap }
```

---

## 2. Canvas Region ("Atelier")

### Intent
Make the canvas feel like a dedicated, well-lit "wall" or work surface — the gallery's centerpiece. Frame it without cluttering it.

### Changes

**Frame treatment:**
- Add subtle inner border to `#dta-canvas-region` using a darker shade to create depth
- Consider a subtle inset shadow (dark only, no glow) to push the canvas area back slightly
- Ensure the canvas element has adequate internal spacing so artwork doesn't touch edges

**Visual hierarchy:**
- Canvas area should feel like the brightest/darkest zone (pure black `#0d0d0d`)
- Add minimal padding inside the canvas region so the canvas element has breathing room
- Ensure canvas maintains aspect ratio without awkward stretching

**Selectors to touch:**
```css
#dta-canvas-region { border, padding }
#dta-canvas { object-fit or sizing adjustments }
```

---

## 3. Sidebar / Controls as Instruments

### Intent
Transform the control groups from "form fields" into approachable, tactile instruments through refined spacing, clearer hierarchy, and satisfying micro-interactions.

### Changes

**Control group refinements:**
- Increase spacing between label and input from `8px` gap to more structured rhythm
- Add subtle left border accent to `.dta-control-group` to visually group controls
- Ensure labels feel like instrument labels — clear but not shouty

**Input/select styling:**
- Add subtle hover state to inputs and selects (border color shift to gold)
- Improve focus states — current outline is good but could be more visible
- Add slight padding increase for more comfortable click targets

**Button hierarchy:**
- **Render button:** Gold `#c9922a` — the primary action, should feel weighty
- **Export button:** Teal `#4a8fa8` — secondary but still important
- Add more tactile active states — current transform is good, enhance with subtle color shift
- Ensure buttons have adequate min-height for comfortable clicking

**Micro-interactions:**
- Inputs: border color transition on hover (0.15s ease)
- Buttons: Keep transform translate + shadow change, add subtle brightness shift on hover
- Selects: Consider subtle background shift on hover

**Selectors to touch:**
```css
.dta-control-group { border-left, padding-left }
.dta-control-group label { margin-bottom adjustments }
.dta-control-group select, .dta-control-group input { hover states, focus states }
.dta-button-row button { min-height, enhanced hover/active states }
```

## 4. Auth Section (Account)

### Intent
Visually differentiate auth as a secondary, quieter concern while keeping it clearly accessible and trustworthy.

### Changes

**Visual demotion:**
- Reduce border prominence — change from `2px solid #2a2a2a` to subtler treatment
- Slightly reduce padding to visually compress the section
- Use muted text color for the summary to indicate "secondary"

**Auth toggle clarity:**
- Enhance `.dta-auth-mode-btn.active` state for clearer distinction
- Add subtle transition between active states
- Ensure inactive buttons look clearly inactive (lower contrast)

**Form calmness:**
- Inputs should feel approachable, not aggressive
- Submit button (teal) should feel trustworthy, not demanding

**Selectors to touch:**
```css
#dta-auth-section { border, padding }
#dta-auth-section summary { color adjustment }
.dta-auth-mode-btn { inactive vs active states }
.dta-auth-mode-btn.active { enhanced visibility }
.dta-auth-form input { border-radius consideration (none per constraints) }
```

## 5. Responsiveness (Light Touch)

### Intent
Handle narrower viewports gracefully without breaking the atelier metaphor. Canvas remains hero; sidebar stays usable.

### Changes

**Breakpoints:**
- **1200px:** Begin reducing sidebar width, allow canvas to grow proportionally
- **1024px:** Minimum comfortable width — sidebar shrinks to `min-width: 260px`, font sizes may reduce slightly

**Adjustments at ≤1200px:**
- Reduce sidebar padding from `20px` to `16px`
- Slightly reduce control group gaps
- Ensure canvas still dominates visually

**Adjustments at ≤1024px:**
- Sidebar `min-width: 260px`, `flex: 0 0 320px`
- Further reduce padding to `12px`
- Ensure scrollbar appears gracefully in sidebar if needed
- Canvas region maintains at least 60% of viewport

**No mobile-first rewrite:** Per constraints, this is desktop-first with pragmatic narrowing only.

**Selectors to touch:**
```css
@media (max-width: 1200px) { #dta-sidebar, .dta-control-group }
@media (max-width: 1024px) { #dta-sidebar, #dta-main }
```

## 6. Micro-interactions and Accessibility

### Intent
Add playful tactility through deliberate, meaningful transitions while ensuring keyboard accessibility is never compromised.

### Changes

**Focus states (accessibility critical):**
- All inputs, selects, buttons must have visible focus indicators
- Current outline style is good — ensure color contrast is sufficient
- Focus outline should be `2px solid #4a8fa8` or `#c9922a` with adequate offset

**Hover states (playful lab):**
- Inputs: border color shifts to gold on hover (0.15s transition)
- Buttons: transform + shadow + subtle brightness increase
- Auth toggle: background shift on hover for inactive buttons

**Active states (tactile feedback):**
- Buttons: current transform/translate approach is good
- Ensure active state feels "pressed" — deeper shadow, slight color darkening

**Transitions:**
- All transitions limited to `0.15s` or `0.2s`
- Properties: `transform`, `box-shadow`, `border-color`, `background-color`, `color`
- No decorative/ambient animations

**Color contrast considerations:**
- Gold `#c9922a` on dark backgrounds: estimated ~6:1 ratio (good)
- Teal `#4a8fa8` on dark backgrounds: estimated ~4.5:1 ratio (acceptable)
- Off-white `#f0ece4` on dark: excellent contrast
- Ensure focus outlines are clearly visible against all backgrounds

**Selectors to touch:**
```css
/* Focus states */
.dta-control-group input:focus, 
.dta-control-group select:focus,
.dta-button-row button:focus,
.dta-auth-form input:focus { outline adjustments }

/* Hover states */
.dta-control-group select:hover,
.dta-control-group input:hover { border-color transition }
.dta-button-row button:hover { enhanced transform/shadow }
.dta-auth-mode-btn:not(.active):hover { background shift }

/* Active states */
.dta-button-row button:active { refined shadow/color }
```
