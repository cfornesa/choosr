# Plan: Fix Featured Artworks - Return All Items + Responsive Grid Layout

## Problem Statement (Final)
When 4 artworks are marked as featured, only 3 display because:
1. The API limits results to 3 via `PORTFOLIO_FEATURED_LIMIT`
2. The CSS needs adjustment to properly handle variable item counts with responsive behavior

**Desired Behavior:**
- Return ALL featured items from API (no arbitrary limit)
- Display as many items per row as fit the screen width
- Mobile: 1 item per row (minimum)
- Desktop: As many as fit, no upper limit
- Items should have consistent minimum width (~220px)

## Root Cause Analysis

### Issue 1: API Hard Limit
File: `config/env.php` line 78
```php
define('PORTFOLIO_FEATURED_LIMIT', 3);
```

File: `api/artworks.php` lines 53-55
```php
if ($limit === null) {
    $limit = PORTFOLIO_FEATURED_LIMIT;  // Always defaults to 3
}
```

### Issue 2: CSS Grid Configuration
Current CSS (`index.php` lines 168-172):
```css
#dta-featured-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 24px;
}
```

The current CSS is actually correct for the desired behavior! The `auto-fit` with `minmax(220px, 1fr)` will:
- Fit as many 220px-minimum columns as possible per row
- Stretch items equally to fill remaining space
- Wrap to new rows when space runs out

## Solution Design

### Part 1: API Changes - Remove Featured Item Limit

Modify `api/artworks.php` to NOT apply a default limit when `filter=featured`. Instead:
- If `limit` parameter is explicitly provided, use it (with max 100 cap)
- If `limit` is NOT provided for featured filter, return ALL featured items

This allows the admin to feature any number of artworks and have them all display.

### Part 2: CSS Refinements - Mobile Single Column

The current CSS already handles the "fit as many as possible" requirement. Only addition needed is a media query to force single column on mobile:

```css
/* Mobile: force single column */
@media (max-width: 500px) {
  #dta-featured-grid {
    grid-template-columns: 1fr;
  }
}
```

### Visual Behavior Summary

| Viewport Width | Items Per Row | Example (4 items) |
|----------------|---------------|-------------------|
| < 500px (mobile) | 1 | 1×4 vertical stack |
| 500-743px | 2 | 2×2 grid |
| 744-967px | 3 | 3+1 (3 top, 1 bottom) |
| 968px+ | 4+ | 4 in one row, or more |

*Calculations based on: item min 220px + 24px gap*

## Safety Considerations

### Upper Bound on Featured Items
The current implementation has no upper limit. If 50 artworks are marked featured, all 50 will display. This is **by design** per user request, but consider:

- **Performance:** 50+ items with images may slow page load
- **UX:** Very long pages may impact user experience

**Mitigation:** The `minmax(220px, 1fr)` ensures items don't get too small, and grid wraps naturally. If this becomes an issue, consider:
1. Adding a soft limit (e.g., max 12) with admin warning
2. Implementing pagination or "Load more" button
3. Adding a scrollable horizontal row option

### Existing `PORTFOLIO_FEATURED_LIMIT` Constant
This constant is only used in the artworks API. Changing the API logic makes this constant unused for featured queries (it's still used for `filter=public` fallback). Options:
1. Keep constant for potential future use
2. Remove constant if truly unneeded
3. Add comment noting it's deprecated for featured filter
