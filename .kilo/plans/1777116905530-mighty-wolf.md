# Plan: Center Art Piece by Default in Manual Dimension Mode

## Problem Statement
In studio.php, the default view does not center the art piece in Manual Dimension mode. The X Position and Y Position should default to center the piece at the canvas center unless further configurations are made.

## Root Cause Analysis

### Current Implementation:
1. `visualDimensions.js` (lines 60-66): Initializes `_values` with `x: 0, y: 0`
2. `controls.js` (lines 200-202): Sets `Controls._currentVisualDimensions = { x: 0, y: 0, ... }`
3. `renderer.js` (lines 597-620): Converts explicit dimensions to normalized values:
   - `normX = ((explicitDimensions.x || 0) + 1) / 2` converts -1..1 to 0..1
   - When X=0: `normX = 0.5`, `translateX = (0.5 - 0.5) * cssWidth = 0`
   - Canvas is translated to `cssWidth / 2 + translateX` = center

### The Issue:
The renderer's grid generation (lines 651-677) creates data points centered around `0.5` coordinate. However, the issue is that **0 is not the conceptual "center" value from the user's perspective** - the slider range is -1 to 1, where 0 represents the middle. The implementation appears correct mathematically, but may not be producing the expected visual result due to how the art styles interpret the position values.

After deeper analysis of the rendering pipeline, the actual issue appears to be:
1. The X/Y position sliders range from -1 to 1, with 0 being the conceptual center
2. The renderer normalizes these to 0-1 range where 0.5 is center
3. The translation calculation is: `translateX = (normX - 0.5) * cssWidth`
4. When X=0 → normX=0.5 → translateX=0 → artwork at canvas center (cssWidth/2)

**However**, the problem is that **the default values in the VisualDimensions UI and the Controls state are 0, but the slider display shows 0 which users don't intuitively understand as "center"**. Additionally, the grid generation uses offsets from 0.5, which when combined with the canvas translation may not produce the desired centered appearance.

## Solution Options

### Option 1: Change Default Values to Explicitly Center (Recommended)
Change the default X and Y values in visualDimensions.js and controls.js from `0` to explicitly calculated center values that ensure proper centering across all art styles.

**Files to modify:**
- `src/controls/visualDimensions.js` (line 61-62): Change `_values` defaults
- `src/controls/controls.js` (line 200-202): Change `_currentVisualDimensions` defaults

**Pros:**
- Simple, focused change
- Directly addresses the user expectation
- Minimal risk of side effects

**Cons:**
- May need adjustment per art style if they interpret position differently

### Option 2: Reframe - Change Slider Range Interpretation
Instead of -1 to 1, change the conceptual model so that X/Y position represents an offset from center in pixels, with default of 0 meaning "no offset" (centered).

**Files to modify:**
- `src/controls/visualDimensions.js`: Change DIMENSION_RANGES for x and y
- `src/canvas/renderer.js`: Update renderUsingExplicitDimensions to handle pixel offsets

**Pros:**
- More intuitive for users (0 = no offset = centered)
- Aligns with how other design tools work

**Cons:**
- Breaking change to existing saved artworks
- Requires more extensive code changes
- Conflicts with C-11 (Artwork Mode Persistence) constraint

### Option 3: Fix Grid Generation Offset Calculation
Modify how the renderer generates grid points to ensure they are truly centered when X=0, Y=0.

**Files to modify:**
- `src/canvas/renderer.js` (lines 651-677): Adjust grid point generation

**Pros:**
- Keeps existing coordinate system
- Targets the visual output directly

**Cons:**
- May vary per art style
- Doesn't address the UI/UX issue of "0" not feeling like center

### Wait - Further Analysis Needed

Looking more carefully at the grid generation in renderer.js (lines 651-677):

```javascript
// Calculate grid position centered at 0.5, 0.5
var gridX = (col / (cols - 1 || 1)) - 0.5;  // -0.5 to 0.5
var gridY = (row / (rows - 1 || 1)) - 0.5;  // -0.5 to 0.5

// Data points centered around (0.5, 0.5) - transforms handle position
var x = 0.5 + gridX * 0.6;
var y = 0.5 + gridY * 0.6;
```

The grid points are generated around the 0.5 coordinate. Then the canvas context is:
1. Translated to `cssWidth/2 + translateX` (where translateX is 0 when X=0)
2. Scaled by `scaleFactor`
3. Rotated by `rotationRad`

The art style's render function receives these data points with x,y around 0.5 and interprets them based on the canvas context transformation.

**The real issue may be:** The grid is generated around 0.5, but when the canvas is translated to the center (cssWidth/2), the art style renders the points relative to that center. If the art style treats 0.5 as a coordinate relative to the canvas center, then the grid spread (±0.3 from 0.5) would produce the expected centered output.

After this analysis, the implementation appears correct. The defaults should already center the artwork. **However**, the user is reporting otherwise, suggesting there may be a discrepancy in how specific art styles interpret the coordinates, or the initial render isn't triggering with the correct defaults.
