## Embedded Assumption to Surface

The handoff prompt specifies that `window.DataToArt.normalize` "already exposes" utilities including `column`, `numeric`, `string`, `boolean`, `date`, `detectType`. **Assumption:** These methods exist and are functional per Session 6 decisions. The normalizer.js module will be a thin wrapper that delegates to these existing utilities. If any method is missing or behaves differently, dataMapper.js's fallback behavior (using sample_values as values) will still function.

---

## Gallery of Implementation Options (Rule 2)

### Option A: Direct Reference Pattern (Recommended)
Controls.js holds direct references to the three sub-modules (ColumnMapper, PalettePicker, Renderer) and calls their methods directly. Simple, transparent, minimal abstraction overhead.

### Option B: Event Bus Pattern
Controls.js publishes events ('datasetChanged', 'paletteChanged') and sub-modules subscribe. More decoupled but adds complexity unnecessary for Phase 1. **Reframe:** Do we need decoupling we don't have a use case for?

### Option C: State Container Pattern
Central state object with getters/setters that automatically trigger renders. Elegant but risks over-engineering. **Unexpected:** Given the "instruments, not forms" design principle, immediate feedback (direct method calls) matches the metaphor better than reactive indirection.

**Decision:** Proceed with Option A. The explicit coordination in controls.js aligns with the workstation metaphor — each instrument has a direct physical relationship to the canvas.

## File 1: src/data/normalizer.js

### Purpose
Thin wrapper module that exposes `window.DataToArt.Normalizer` as a convenience interface delegating to the existing `window.DataToArt.normalize` utilities established in Session 6.

### Public API
```javascript
window.DataToArt.Normalizer = {
  column: fn,      // delegates to window.DataToArt.normalize.column
  numeric: fn,     // delegates to window.DataToArt.normalize.numeric
  string: fn,      // delegates to window.DataToArt.normalize.string
  boolean: fn,     // delegates to window.DataToArt.normalize.boolean
  date: fn,        // delegates to window.DataToArt.normalize.date
  detectType: fn,  // delegates to window.DataToArt.normalize.detectType
  normalizeDataset: fn  // NEW — iterates columns, adds normalized_values
}
```

### Implementation Details

**Dependency Check:**
- Verify `window.DataToArt.normalize` exists before wrapping
- If missing, log warning (DEBUG flag) and provide no-op fallbacks

**normalizeDataset(dataset) Algorithm:**
1. Create shallow copy of dataset object (preserve immutability)
2. Iterate `dataset.columns` array
3. For each column:
   - Call `window.DataToArt.normalize.column(column.values, column.data_type)`
   - Add `normalized_values` property to column object
   - Normalized values are floats in range [0,1] or null for unmapped/missing
4. Return modified dataset copy

**Return Shape:**
```javascript
{
  dataset_id: int,
  columns: [
    {
      column_name: string,
      display_name: string,
      data_type: string,
      values: array,
      sample_values: array,
      normalized_values: array  // ← ADDED by normalizeDataset
    }
  ],
  row_count: int
}
```

## File 2: src/data/dataMapper.js

### Purpose
Transforms raw PHP API responses into the renderer-ready dataset shape. Handles JSON parsing, column hydration, and fallback population.

### Public API
```javascript
window.DataToArt.DataMapper = {
  mapApiResponse: fn,      // apiResponse → dataset object
  mapColumnToVisual: fn    // (column, dimensionKey) → mapping entry
}
```

### Implementation Details

**mapApiResponse(apiResponse) Algorithm:**
1. Input validation: verify apiResponse is object with `dataset_id` and `columns` array
2. Create base dataset object with `dataset_id`, `row_count`
3. Transform columns array:
   - For each column in apiResponse.columns:
     - Parse `sample_values` if it's a JSON string (PHP sometimes encodes arrays as JSON)
     - Parse `values` if present and JSON string
     - **Fallback logic:** If `values` is missing or empty, populate from `sample_values`
       ```javascript
       // COMMENT: Using sample_values as preview-only data fallback
       // Full dataset values should be provided by API for production use
       ```
     - Ensure `data_type` is present (default to "unknown" if missing)
     - Return normalized column object
4. Return complete dataset object matching Session 6 contract

**mapColumnToVisual(column, dimensionKey) Algorithm:**
1. Returns object: `{ [dimensionKey]: column.column_name }`
2. Used by controls.js to build columnMapping objects

**Input Handling (PHP API quirks):**
- PHP may return arrays as JSON strings — detect string type and `JSON.parse()`
- Handle `null` or `undefined` values gracefully
- Defensive: if `columns` is missing, return null and log warning (DEBUG)

**Return Shape:** Identical to Session 6 dataset contract

## File 3: src/controls/columnMapper.js

### Purpose
Renders column-to-dimension mapping UI as instrument-like controls. Provides dropdowns for each visual dimension populated from dataset columns.

### Visual Design (per DESIGN.md + CONSTRAINTS.md)
- Container: surface background (#1a1a1a), border (#2a2a2a)
- Labels: system-ui font, off-white text (#f0ece4)
- Select dropdowns: dark input style, monospace for data values
- No gradients, hard offset shadows (4px 4px 0px) if elevated
- Layout: vertical stack of dimension rows

### Public API
```javascript
window.DataToArt.ColumnMapper = {
  render: fn,      // (containerEl, dataset, currentMapping, onChangeCallback)
  getMapping: fn,  // → current columnMapping object
  reset: fn        // clears all dimension assignments
}
```

### Implementation Details

**Visual Dimensions (fixed set):**
- `x` — horizontal position
- `y` — vertical position  
- `size` — element size/radius
- `color` — color index mapping
- `opacity` — transparency
- `rotation` — angular rotation

**render() Algorithm:**
1. Clear containerEl innerHTML
2. Create header: "Map Data to Visual Dimensions"
3. For each dimension in [x, y, size, color, opacity, rotation]:
   - Create row container (div)
   - Create label element (text: dimension name)
   - Create select element:
     - Add blank option at top: `<option value="">— none —</option>`
     - Populate with dataset.columns.map(c => c.column_name)
     - Pre-select currentMapping[dimension] if exists
   - Add change listener:
     - Update internal mapping state
     - Call onChangeCallback with full updated mapping object
   - Append to container

**getMapping() Algorithm:**
- Return current internal state object
- Shape: `{ x: "column_name"|null, y: "column_name"|null, ... }`

**reset() Algorithm:**
- Set all dimension values to null
- Re-render dropdowns with blank selected
- Trigger onChangeCallback with empty mapping

**State Management:**
- Internal `_mapping` object holds current assignments
- Callback fires on every change with complete mapping (not delta)

### Purpose
Renders palette selection UI with curated dark-compatible palettes and custom color mode. Background color selector included.

### Visual Design (per DESIGN.md)
- Container: surface background (#1a1a1a)
- Palette swatches: color squares in a row
- Selected state: hard offset shadow or border accent
- Custom mode: hex input fields, up to 8 colors
- Background selector: color input or hex text field

3. **Plasma** (purple-pink)

4. **Magma** (purple-black)
   - `['#000004', '#180f3d', '#440f76', '#721f81', '#9e2f7f', '#cd4076', '#f1605d', '#fea16e']`

5. **Cividis** (colorblind-friendly)

6. **Fornesus** (abstract/emotional — from fornesusart.com palette)

### Public API
```javascript
window.DataToArt.PalettePicker = {
  render: fn,      // (containerEl, currentPalette, onChangeCallback)
  getPalette: fn,  // → current paletteConfig object
  reset: fn        // returns to first built-in palette
}
```

### Implementation Details

**render() Algorithm:**
1. Clear containerEl innerHTML
2. Create header: "Color Palette"
3. Create curated palettes section:
   - For each built-in palette:
     - Create swatch container (row of color squares)
     - Add click handler: select this palette
     - Show selected state (border or shadow)
4. Create custom palette section:
   - Toggle button: "Custom..."
   - When active: show 8 hex input fields
   - "Add Color" button (disabled at 8 colors)
   - "Remove" button per color
5. Create background color selector:
   - Label: "Canvas Background"
   - Default: #0d0d0d
   - Color input or text field
6. On any change: call onChangeCallback with:
   ```javascript
   {
     colors: ['#hex', '#hex', ...],
     background: '#0d0d0d'
   }
   ```

**getPalette() Algorithm:**
- Return current internal state
- Shape: `{ colors: string[], background: string }`

**reset() Algorithm:**
- Select first built-in palette (Inferno)
- Reset background to #0d0d0d
- Re-render
- Trigger callback

**State Management:**
- Internal `_palette` object holds colors array and background
- Internal `_mode` tracks 'built-in' or 'custom'
- Internal `_selectedIndex` tracks which built-in is selected

## File 5: src/controls/controls.js

### Purpose
Top-level coordinator for the entire control surface. Manages application state, initializes sub-modules, and orchestrates renderer calls.

### Architecture Pattern
Controls.js acts as the **conductor** — it holds references to all instruments (ColumnMapper, PalettePicker, Renderer) and coordinates their interactions. It does not render UI itself; it delegates to specialized modules.

### State Managed
```javascript
{
  currentDataset: object|null,
  currentColumnMapping: object,
  currentPaletteConfig: object,
  currentRenderingConfig: object,
  currentStyleKey: string
}
```

### Public API
```javascript
window.DataToArt.Controls = {
  // Initialization
  init: fn,              // (canvasElement, controlsContainerElement, options)
  
  // State setters (all trigger debounced render)
  loadDataset: fn,       // (dataset) → re-renders columnMapper, triggers render
  setStyle: fn,          // (styleKey) → updates style, triggers render
  setRenderingConfig: fn,// (rc) → merges config, triggers render
  
  // Actions
  triggerRender: fn,     // calls renderer.render() with current state
  triggerExport: fn,     // (filename) → calls renderer.exportPNG()
  reset: fn              // clears all state, clears canvas
}
```

### Implementation Details

**init(canvasEl, controlsEl, options) Algorithm:**
1. Store references to canvas and controls container
2. Validate dependencies:
   - Check `window.DataToArt.Renderer` exists
   - Check `window.DataToArt.ColumnMapper` exists
   - Check `window.DataToArt.PalettePicker` exists
   - If any missing, log warning (DEBUG) and return early
3. Initialize state with defaults:
   - `currentColumnMapping = {}`
   - `currentPaletteConfig = { colors: [...], background: '#0d0d0d' }`
   - `currentRenderingConfig = options.renderingConfig || { animate: false }`
   - `currentStyleKey = options.styleKey || 'particle_field'`
4. Initialize Renderer:
   - `this._renderer = new window.DataToArt.Renderer(canvasEl, options.rendererOptions)`
5. Initialize ColumnMapper:
   - Create container div inside controlsEl
   - Call `ColumnMapper.render(container, null, {}, this._onMappingChange.bind(this))`
6. Initialize PalettePicker:
   - Create container div inside controlsEl
   - Call `PalettePicker.render(container, null, this._onPaletteChange.bind(this))`
7. Set up debounced render trigger (150ms)

**Debounced Render Pattern:**
```javascript
this._renderPending = false;
this._debounceTimer = null;

_triggerRenderDebounced() {
  if (this._debounceTimer) clearTimeout(this._debounceTimer);
  this._debounceTimer = setTimeout(() => {
    this.triggerRender();
  }, 150);
}
```

**loadDataset(dataset) Algorithm:**
1. Validate dataset shape (dataset_id, columns array)
2. Store: `this._currentDataset = dataset`
3. Re-render ColumnMapper with new dataset:
   - `ColumnMapper.render(columnContainer, dataset, this._currentColumnMapping, this._onMappingChange.bind(this))`
4. Call `_triggerRenderDebounced()`

**_onMappingChange(newMapping) Algorithm:**
1. Store: `this._currentColumnMapping = newMapping`
2. Call `_triggerRenderDebounced()`

**_onPaletteChange(newPalette) Algorithm:**
1. Store: `this._currentPaletteConfig = newPalette`
2. Call `_triggerRenderDebounced()`

**setStyle(styleKey) Algorithm:**
1. Validate styleKey exists in `window.DataToArt.ArtStyles.listStyles()`
2. Store: `this._currentStyleKey = styleKey`
3. Call `_triggerRenderDebounced()`

**setRenderingConfig(rc) Algorithm:**
1. Merge rc into `this._currentRenderingConfig` (shallow merge)
2. Call `_triggerRenderDebounced()`

**triggerRender() Algorithm:**
1. If no dataset loaded, call `this._renderer.clear(this._currentPaletteConfig.background)` and return
2. Build rendering parameters:
   ```javascript
   this._renderer.render(
     this._currentDataset,
     this._currentColumnMapping,
     this._currentPaletteConfig,
     this._currentRenderingConfig,
     this._currentStyleKey
   );
   ```

**triggerExport(filename) Algorithm:**
1. If no dataset loaded, log warning and return
2. Call `this._renderer.exportPNG(filename)`

**reset() Algorithm:**
1. Clear all state:
   - `this._currentDataset = null`
   - `this._currentColumnMapping = {}`
   - `this._currentPaletteConfig = { colors: [...], background: '#0d0d0d' }`
   - `this._currentRenderingConfig = { animate: false }`
   - `this._currentStyleKey = 'particle_field'`
2. Call `ColumnMapper.reset()`
3. Call `PalettePicker.reset()`
4. Call `this._renderer.clear('#0d0d0d')`

```
1. particleField.js      → defines ParticleFieldStyle

3. flowingCurves.js      → defines FlowingCurvesStyle

9. palettePicker.js      → pure UI, exposes PalettePicker
10. controls.js          → uses all above, exposes Controls
11. app.js               → initializes Controls, wires to app
```

**Dependency Safety:**
- Each module checks for dependencies before use
- Missing dependencies logged via DEBUG flag, soft fail
- No circular dependencies — tree structure maintained

## Summary

This plan specifies five JavaScript modules that complete the UI controls layer and data pipeline:

1. **normalizer.js** — Thin wrapper around existing `window.DataToArt.normalize`, adds `normalizeDataset()` for column batch processing
2. **dataMapper.js** — Transforms PHP API responses to renderer-ready dataset shape, handles JSON parsing and fallback values

4. **palettePicker.js** — Renders palette selection with 6+ curated palettes + custom mode, includes background selector

All modules follow the established patterns:
- IIFE wrapper, `window.DataToArt` namespace
- No ES modules, no external libraries
- DEBUG flag for logging
- Graceful dependency checking
- Dark atelier visual design (no gradients on UI, hard shadows if elevated)
