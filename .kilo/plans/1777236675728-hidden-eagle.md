# exhibit.php `<main>` Width Fix — CSS Only

## Issue
The `#dta-exhibit-main` element renders at ~200% screen width on mobile because the inline `<style>` block in `exhibit.php` (line 172-177) sets `max-width: 1000px`, which constrains the content to a fixed width larger than the viewport on small screens.

## Root Cause
| Location | Rule | Effect |
|----------|------|--------|
| exhibit.php line 174 | `max-width: 1000px` | Fixed width causes overflow on mobile |
| app.css lines 988-991 | `max-width: 100%` added in 768px media query | **Overridden** by inline styles loading after app.css |

Inline `<style>` blocks in HTML take precedence over external `app.css` because they appear later in the document and have equal specificity.

## Assumption
The inline `<style>` block in exhibit.php (lines 136-317) is the source of truth for exhibit-specific styles, and the `max-width: 1000px` on `#dta-exhibit-main` is causing the ~200% width on mobile.

## Solution
Add a mobile-specific media query inside exhibit.php's inline `<style>` block (around line 317, before the closing `</style>` tag) that overrides `#dta-exhibit-main` with `max-width: 100%` and reduced padding.

## Implementation
In `exhibit.php`, add before `</style>` (line 317):

```css
@media (max-width: 768px) {
  #dta-exhibit-main {
    max-width: 100%;
    padding: 16px;
  }
}
```

## Scope
- **Only** this CSS rule changes in exhibit.php's inline `<style>` block
- **No other exhibit.php CSS** touched
- **No app.css changes** (app.css rule remains as documentation/backup)

## Files Affected
| File | Change |
|------|--------|
| `exhibit.php` | Add mobile override for `#dta-exhibit-main` in inline `<style>` block |

## Verification
1. Load exhibit.php on mobile viewport (≤768px)
2. `#dta-exhibit-main` should fill viewport width with `max-width: 100%`
3. Padding should be reduced to 16px (vs 32px 24px on desktop)
4. No horizontal scrollbar on mobile
