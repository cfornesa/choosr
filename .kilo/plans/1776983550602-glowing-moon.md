## 3. Input Validation Rules

### 3.1 register.php

| Field | Rule | Error Message |
|-------|------|---------------|
| username | 3-64 characters, alphanumeric + underscore only (`/^[a-zA-Z0-9_]+$/`) | "Username must be 3-64 characters and contain only letters, numbers, and underscores" |
| email | `filter_var($email, FILTER_VALIDATE_EMAIL)` | "Please provide a valid email address" |
| password | Minimum 8 characters | "Password must be at least 8 characters long" |
| username (unique) | Check against DB before insert | "Username already taken" |
| email (unique) | Check against DB before insert | "Email address already registered" |

### 3.2 login.php

| Field | Rule | Error Message |
|-------|------|---------------|
| email | Required, non-empty | "Invalid credentials" (generic) |
| password | Required, non-empty | "Invalid credentials" (generic) |

**Security Note:** Both email and password validation failures return the same generic "Invalid credentials" message to prevent user enumeration attacks.

## 4. Password Hashing Approach

- **Algorithm:** `PASSWORD_BCRYPT` (via PHP's `password_hash()`)
- **Verification:** `password_verify()` for checking against stored hash
- **Rationale:** Industry standard, automatic salt generation, future-proof via `password_needs_rehash()` if needed

## 5. Session Configuration Strategy

### 5.1 Session Constants (from env.php)

```php
SESSION_NAME = 'data_to_art_session'
SESSION_LIFETIME = 86400  // 24 hours
```

### 5.2 Session Configuration (applied in login.php and session.php)

```php
ini_set('session.cookie_httponly', 1);
ini_set('session.use_only_cookies', 1);
ini_set('session.cookie_samesite', 'Lax');
ini_set('session.gc_maxlifetime', SESSION_LIFETIME);
session_name(SESSION_NAME);
session_start();
```

### 5.3 Session Data Structure

```php
$_SESSION['user_id']    = (int) $user['id'];
$_SESSION['username']   = (string) $user['username'];
$_SESSION['created_at'] = time(); // For potential session age checking
```

## 7. session.php Usage Pattern

### 7.1 How Other Endpoints Use session.php

```php
<?php
// At the top of any protected API endpoint
require_once __DIR__ . '/auth/session.php';

// If execution continues, $currentUserId and $currentUsername are available
// The session.php file will exit with 401 if auth fails
```

### 7.2 session.php Behavior

| Condition | Action |
|-----------|--------|
| Valid session exists | Start session, populate `$currentUserId` and `$currentUsername`, allow script to continue |
| No session / expired | Output `{"success":false,"error":"Authentication required"}`, HTTP 401, `exit` |
| Invalid session data | Same as above |

### 7.3 Protected Endpoints (Future)

- `api/artwork/save.php` — Requires valid session
- `api/artwork/list.php` — Requires valid session  
- `api/datasets/user.php` — Requires valid session

## 8. File Specifications

### 8.1 api/auth/register.php

**Method:** POST only (return 405 for other methods)

**Input:** JSON body `{ username, email, password }`

**Processing Flow:**
1. Validate HTTP method
2. Parse JSON input
3. Validate username format (regex)
4. Validate email format (filter_var)
5. Validate password length (>= 8)
6. Check username uniqueness (SELECT query)
7. Check email uniqueness (SELECT query)
8. Hash password with `password_hash($password, PASSWORD_BCRYPT)`
9. Insert into users table with PDO prepared statement
10. Return success with new user_id

**Error Cases:**
- Invalid JSON → 400
- Missing fields → 400  
- Invalid username format → 400
- Invalid email format → 400
- Password too short → 400
- Username taken → 400
- Email taken → 400
- DB error → 500

### 8.2 api/auth/login.php

**Method:** POST only (return 405 for other methods)

**Input:** JSON body `{ email, password }`

**Processing Flow:**
1. Validate HTTP method
2. Parse JSON input
3. Fetch user by email (SELECT with prepared statement)
4. Check if user exists and is_active = 1
5. Verify password with `password_verify()`
6. Configure and start session
7. Store user_id and username in $_SESSION
8. Return success with username

**Error Cases:**
- Invalid JSON → 400
- Missing fields → 400
- User not found → 401 (generic "Invalid credentials")
- User inactive → 401 (generic "Invalid credentials")
- Password mismatch → 401 (generic "Invalid credentials")
- DB error → 500

### 8.3 api/auth/session.php

**Type:** Utility file (not a standalone endpoint)

**Usage:** `require_once __DIR__ . '/auth/session.php'`

**Processing Flow:**
1. Configure session security settings
2. Set session name from constant
3. Start session
4. Check for valid `$_SESSION['user_id']`
5. If invalid/expired: output JSON error, HTTP 401, exit
6. If valid: expose `$currentUserId` and `$currentUsername`

**Exposed Variables:**
- `$currentUserId` — (int) The authenticated user's ID
- `$currentUsername` — (string) The authenticated user's username

## 9. Security Considerations

| Concern | Mitigation |
|---------|------------|
| SQL Injection | PDO prepared statements for all queries |
| Password storage | bcrypt hashing via `password_hash()` |
| Session fixation | `session.use_only_cookies = 1` |
| XSS via session cookie | `session.cookie_httponly = 1` |
| CSRF (future) | Token-based CSRF protection to be added in Phase 2 |
| User enumeration | Generic error messages on login failure |
| Timing attacks | `password_verify()` provides constant-time comparison |
